RBAC Architecture Overview

Role-Based Access Control implemented via Django Groups (Lab 6)

Enterprise Security Standard

Passwords are never hardcoded in source code. All credentials are securely stored in the database (auth_user table) with automatic hashing. Authentication is handled dynamically via Django's authenticate() function, which compares user input against the hashed password at runtime.

How Authentication Works

Step 1 · Input

User submits credentials via the login form. Password is read dynamically from request.POST.get("password")

Step 2 · Verify

authenticate() compares the input against the hashed password stored in the auth_user table.

Step 3 · Redirect

AccountService checks the user's Django Group and redirects to the appropriate role-based dashboard.
